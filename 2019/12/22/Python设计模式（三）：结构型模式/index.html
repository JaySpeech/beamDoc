Python设计模式（三）：结构型模式

发表于 2019-12-22 更新于 2019-12-23 分类于 编程 阅读次数： Valine：
本文字数： 7.4k 阅读时长 ≈ 7 分钟
创建型模式搞完了，开始结构型模式，包括适配器模式、修饰器模式、外观模式、享元模式、模型视图控制器模式和代理模式

结构型模式处理一个系统中不同实体之间的关系，关注的是提供一种简单的对象组合方式来创造新功能。

适配器模式

适配器模式帮助我们实现两个不兼容接口之间的兼容。说白了就是想办法，让不同类不同的接口名称，在另外一个类中统一由一个接口实现，这样无需修改不兼容模型的代码就能获得接口的一致性。

修饰器模式

修饰器模式，就是借用python中的语法糖，扩充函数的功能，比如内存占用输出，执行时间输出，log打印等，这种方法无需通过继承或者组合就能任意扩展对象的行为。这种方法，可参照python语法修饰器章节，这里不多赘述。

外观模式

外观模式，有助于隐藏系统内部的复杂性，通过一个简单的接口向客户暴露必要的部分。本质上，外观上在已有的复杂系统上实现一个抽象层。

计算机是一个复杂的机器，全功能依赖多个复杂的部件。但是这些内部的操作不会暴露给客户端，而是创建一个外观来封装整个过程，并保证所有步骤按次序执行。

享元模式

享元模式通过为相似对象引入数据共享来最小化内存使用，提升性能。一个享元就是一个包含独立状态的不可变数据对象。使用时要满足以下三个条件：

1.应用需要使用大量的对象

2.对象太多，创造代价太大。

3.对象ID对于应用不重要。共享对象会造成对象同一。

模型-视图-控制器模式

MVC模式应用面向对象的的Soc原则，把软件切分为三个主要部分，即模型、视图、控制器三个部分。每个部分都有明确的分工，模型负责访问数据，管理应用的状态；视图是模型外在的表现，且不一定是图形化的；控制器则是模型与视图之间的连接。MVC恰当使用，能确保程序的易于维护、易于扩展。

如果不确定自己写的代码能否满足MVC，请问自己：

1.你的应用有GUI，能换肤嘛？换起来麻烦嘛？

2.没有GUI，添加起来麻烦嘛？
