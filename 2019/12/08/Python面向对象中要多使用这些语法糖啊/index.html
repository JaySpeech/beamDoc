Python面向对象中要多使用这些语法啊！

发表于 2019-12-08 分类于 编程 阅读次数： Valine：
本文字数： 4.8k 阅读时长 ≈ 4 分钟
Python面向对象很强大，不能仅仅局限于一些基本功能啊！迈开步子朝前跨，以后多用这些高级功能，很赞！

Python写了也有一年多了，谈不上精通，也算是熟练了。从最开始的瞎写到面向过程再到最近的面向对象，确切地感受到了Python的强大。前一段时间学习C++，由于是系统的学习，接触到了很多面向对象的语法，如菱形继承、成员函数重载运算符等功能时，惊呼C家族的面向对象竟然这么牛逼，连Python中没有的它都有！

是真的没有嘛？当然不是了，只是我之前没有了解接触过而已，所以造成了没有这种语法功能的错误。这也和我入门学习Python的路径有关，直接看了基本语法之后上手，且在后续的项目中只围绕这些基本语法展开实战，没有及时地补充新鲜知识从而导致此之现状。所以入门并快速熟练使用一门语言的最佳路径应该是：熟悉完基本语法后快速上手实战的同时继续补充稍微高级一点的语法，然后再实战，如此反复，直到精通。

周天在家刷油管视频，看到有一门叫Python OOP Tutorials的教程就想着随便点进去看看，结果这不是发现了新大陆了。

类方法和静态方法

在早之前接触过类方法和静态方法，也在自己编写的静态方法中用过staticmethod，和static variables一样是类所有实例共享的，不能改变类成员的属性。作为与类有关联的函数，在类中定义了，类似于定义了一个局部域函数为该类专门服务。见过许多面向对象的写法是先定义个staticmethod然后再定一个self的普通的method，达到改变类状态的功能以方便使用。

我理解这种方法非常类似于C++中内置函数的重载，比如加减乘除等等在新建的类中完成定义后，可以像内置类型一样运用搞得飞起。比如像str()\repr()等等。

引用 str() vs repr() in Python中的一段话

Following are differences:

  • str() is used for creating output for end user while repr() is mainly used for debugging and development. repr’s goal is to be unambiguous and str’s is to be readable. For example, if we suspect a float has a small rounding error, repr will show us while str may not.
  • repr() compute the “official” string representation of an object (a representation that has all information about the abject) and str() is used to compute the “informal” string representation of an object (a representation that is useful for printing the object).
  • The print statement and str() built-in function uses str to display the string representation of the object while the repr() built-in function uses repr to display the object.

str()方法呢是给最终的用户，告诉大家返回了一个什么玩意，而repr呢则更底层一些告诉了这个对象是如何产生的，如果不使用这些方法呢，打印出来就是_xxxdad_, objectxxx之类的乱七八糟的妖魔鬼怪。
